# PALINDA, quicksort och projekt

## Quicksort

Gör [quicksort](https://kth.kattis.com/courses/DD1338/algdat25/assignments/o693ti/problems/kth.alginda.quicksort)-uppgiften
på Kattis. Det går bra att använda valfritt programmeringsspråk. Det går även
bra att använda någon annan algoritm. Försök få en så snabb lösning som möjligt!
Quicksort ska vara gjord på fredagen andra veckan av tentaperioden, alltså
**13/3**.

## PALINDA

Skapa en kopia av [palinda](https://github.com/INDA25PlusPlus/palinda)-repot i INDA25PlusPLus
med namnet `<kth_id>-palinda`. Repot ska vara **privat**!

Ni kan t.ex. göra så här:

1. Skapa ert eget repo.

2. Kör dessa kommandon:

   ```bash
   $ git clone https://github.com/INDA25PlusPlus/palinda
   $ git -C palinda remote set-url origin länk_till_ert_repo
   $ git -C palinda push -u origin main
   ```

Uppgifterna ska vara lösta absolut senast **27/3**, men helst innan nästa
övning, så ni kan fokusera på projektet.

[Här](https://go.dev/tour/list) är en genomgång av språket Go som används i PALINDA:n.

## Projekt

Till nästa övning ska ni inom projektgruppen försöka specificera
projektet så nogrannt som möjligt. Ni ska skriva en rapport på
ca. 1 sida samt förbereda en presentation baserat på detta.

Rapporten/presentationen ska innehålla:

* Vad är projektet?
  * Vad är er MVP (minimum viable product)
  * Vilka ytterligare features vill implementera om det finns tid?

* Hur fördelas arbetet?
  * Försök dela in projektet i moduler (t.ex. rendering/hasning etc.)
  * Vem/vilka ska göra vad?
  * Vad tror ni tar mest tid?

* Hur ska arbetet se ut?
  * Schemalagda möten?
  * Version control? (PR:s, rebase/merge etc.)

* Vad ska ni använda för teknologi?
  * Programemringsspråk?
  * Bibliotek?
  * Coding guidelines?

Det absolut viktigaste är att alla kan delta under hela projektets gång.
